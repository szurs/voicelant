Voicelant Plugins

Плагины Voicelant

Особенностью Voicelant является возможность добавления дополнительной функциональности через использование программных модулей, написанных сторонними поставщиками - плагинов. Для написания таких модулей Voicelant публикует программный интерфейс. Такие команды Voicelant, как Database, VBScript, DLL реализованы с помощью плагинов, входящих в комплект поставки. Набор плагинов для системы зависит от конкретных задач. Дополнительные плагины можно подключать по мере небходимости. Основные плагины входят в поставку Voicelant, другие поставляются по желанию заказчика.

Таким образом Voicelant функционально можно разделить на телефонную часть, которая предоставляет функции по обработке вызовов и часть, которая производит обработку данных с помощью плагинов.

Примеры плагинов для расширения - различные способы доступа к базам данных (OLE DB, ODBC, фирменные интерфейсы), HTTP, FTP, Telnet доступ к данным, отправка и получение e-mail, SMS, Text-To-Speech (TTS), проприетарные интерфейсы, нестандартные источники данных (редкие БД, системы промышленной автоматизации - датчики). При развитой системе плагинов Voicelant может даже не использовать телефонную часть в своей работе, а заниматься только обработкой и диспетчеризацией данных.

Плагины хранятся в специальном подкаталоге системы \PLUGINS и представляют собой файлы DLL, имеющие расширение pvt, которые подключаются при запуске приложения ServerVT. Взаимодействие с плагином при обработке вызовов во время выполнения программы ServerVT осуществляет с помощью команды Plugin.

Процесс взаимодействия Voicelant с плагинами можно разделить на три этапа:

1.Установка плагина  - это помещение файла плагина в подкаталог .\PLUGINS системы Voicelant.
2.Подключение плагина осуществляется  при запуске программы ServerVT.
3.Вызов плагина происходит из сценария по имени сервиса во время работы Voicelant.

 

Плагины бывают стандартные и нестандартные. Стандартные плагины могут иметь файл <имя_плагина>.ini в котором содержатся основные параметры настройки плагина:

·<Path Plugin> - путь до файла плагина.
·<Service name> - имя сервиса для вызова плагина из сценария при работе Voicelant. Некоторые плагины (например, MSSQL.pvt) могут иметь несколько сервисов с разными именами (MSSQL и MS-SQL) и параметрами, например, для того, чтобы разделить запросы к различным источникам данных, имеющим разное время отклика;
·<Connection string> - дополнительная строка параметров, используемая при подключении плагина (может быть пустой);
·<Count threads> - количество программных потоков, которые одновременно могут обрабатывать запросы к данному  сервису. Например, для плагина MSSQL.pvt - это количество соединений с базой данных MS-SQL Server одновременно поддерживаемых при работе Voicelant для сокращения времени отклика.

Пример содержимого файла MSSQL.ini для плагина MSSQL.PVT:

 

[MSSQL]

ConnectionString=File Name=mssql.udl

CountThreads=2

[MS-SQL]

ConnectionString=File Name=mssql.udl

CountThreads=2

 

Если стандартный плагин не имеет файла <имя_плагина>.ini, то подразумевается, что данный плагин использует только внутренние настройки и получает при подключении только один поток для выполнения.

 

Нестандартные плагины внутри себя содержат окно (диалог) для настройки необходимых специфических параметров, которое вызывается так же как и для стандартных плагинов.

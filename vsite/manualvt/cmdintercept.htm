Перехват канала (Intercept)

 Top  Previous  Next

Команда <Intercept> производит поиск канала по заданным критериям и сохраняет ссылку на него. Далее ссылка на найденный канал может использоваться командами - для выполнения каналом нового задания (например, сценария исходящего вызова) и - для связывания двух каналов (своего и найденного).

 

 

В левой части окна  - возможные критерии для поиска. В правой части окна  - набор выбранных критериев для поиска.

 

Выбор и удаление критериев производится кнопками ">" и "<". Данные для выбранного критерия можно задать кнопкой <Data>.

 

·<CHAN_TYPE> - тип канала: IN - входящий канал, OUT - исходящий канал, INOUT - работает в обоих направлениях, UNDEF - неопределено.
·<CHAN_SELECT> - выбор канала: ANY - любой канал.
·<CHAN_STATE> - текущее состояние канала: IDLE - канал свободен, CONNECT - канал в состоянии проключенного вызова, UNDEF - неопределено.
·<CALL_DIRECTION> - направление вызова (только для каналов в состоянии вызова): IN - входящий вызов, OUT - исходящий вызов, UNDEF - неопределено.
·<CALLING_NUM> - номер вызывающего абонента (только для каналов в состоянии вызова).
·<CALLED_NUM> - номер вызываемого абонента (только для каналов в состоянии вызова).
·<REDIR_NUM> - номер, с которого была переадресация (только для каналов в состоянии входящего вызова и при наличии соответствующей возможности сигнализации).
·<CALL_CATEGORY> - категория вызывающего абонента (только для каналов в состоянии входящего вызова и при наличии соответствующей возможности сигнализации)."10" - категория "обычного" абонента.

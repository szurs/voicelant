Work of system Voicelant is defined of scenario that implements the desired functionality for an incoming call or initiating outbound connections.

 

The scenario represents a program written in simple script-like language, which has the following properties:

 

· small number of commands and data types;
· the possibility of branching and looping;
· use variables;
· orthogonal syntax is oriented to the subject area;
· possibility to save the local fragments of script in files and connect them as required;
· possibility to use external modules (plugins) to access remote resources (databases, e-mail, Internet, other software blocks);

 

Write a script easily using VisualVT.

 

Scripts are stored in files with the extension .pat in subdirectory \PATTERN.

 

When an incoming call for all channels is loaded the script from a file that is defined when configuring Voicelant (TuningVT).

Outgoing calls are made as follows: the channels which have the possibility of handling outgoing calls after a specified timeout load the script from the file for outgoing connections, which checks the availability of tasks for autodialing by using of corresponding script commands (such as a sample from the database) and with the necessary information, perform the command , which makes dialing and establishes an outgoing connection (hereinafter - the scenario).

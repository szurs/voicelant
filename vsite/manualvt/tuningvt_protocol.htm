Tab <Protocol> - setting options for protocols Voicelant.

· <Hard protocol> - If checked, the system Voicelant writes log for incoming and outgoing calls. Logs are produced for each channel and stored in text files with names like "YYYYMMDD_HHMMSS_XXX. Log" in subdirectory \PROTOCOL, where YYYYMMDD - year, month, date (creation date), HHMMSS - hours, minutes, seconds (time of creation), XXX - channel number. Each call is recorded in the file as a string "DNIS: 648923 ANI: 3422900197 Date: 11/19/04 Time: 11:56:15 N: * N: 3294 # DISC: NET DateEnd: 11/19/04 TimeEnd: 11: 59:41, where DNIS: 648923 - the destination number, ANI: 3422900197 - the number of the caller, Date: 11/19/04 Time: 11:56:15 - Date and time of the call, N: * N: # 3294 - DTMF codes, entered by caller
·<Enter max size of protocol for each channel (Mb)> - the maximum size of the log file for the achievement of which creates a new file.
· <Test protocol> - If checked, the system keeps a test protocol, which can be used to detect abnormal situations in the telephone network. In normal operation, this log must be turned off.

 

       In addition, the system can keep a record of incoming and outgoing calls in the form of records in the database table (for which there is access through plugin). Maintaining such a protocol is defined by the relevant scenario.

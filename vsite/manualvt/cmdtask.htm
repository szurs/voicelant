Задание (Task)

 Top  Previous  Next

Команда <Task> посылает на выполнение выбранному каналу новое задание в виде сценария.

 

 

·<Variables> - закладка для установки переменных, передаваемых в сценарий задания.
·<Task source> - закладка для установки сценария задания.
·<Wait 20 (s)> или <Not wait> - ждать (сколько секунд) или не ждать окончания выпонения задания.
·<Send task to "OUT" channel> - установка получателя задания: OUT - поиск свободного канала для выполнения исходящего вызова; INTERCEPTED - ; EXTERNAL - для внешнего канала.

 

При получении задания выбранный удаленный канал начнет исполнять переданный сценарий, при этом все переменные, установленные в задании будут ему доступны.

 

Если задано ждать N > 0 секунд окончания выполнения задания, то возможны следующие варианты:

1.Если получатель задания OUT - канал, то текущий сценарий своего канала будет ожидать результат выполнения команды набора номера и установления исходящего соединения для канала, выполняющего задание.  Результат выполнения задания можно увидеть в <Record> с номером 0 или в переменной "RETCODE". Возможные значения: CONNECT - исходящее соединение удаленным каналом установлено, BUSY - абонент занят, NOANSWER - нет ответа.

2.Если получатель задания INTERCEPTED - канал, который уже был в состоянии вызова, то текущий сценарий своего канала будет ожидать окончания вызова для канала, выполняющего задание. Результат выполнения задания удаленным каналом также можно посмотреть в <Record> с номером 0 или в переменной "". Возможные значения: NORMAL - нормальное завершение вызова, BUSY - занято.
3.Если отведенный тайм-аут истек, то свой канал начнет выполнять сценарий, ранее загруженный командой  - <Scenario after timeout> или вызов завершится, если такой сценарий не загружен.

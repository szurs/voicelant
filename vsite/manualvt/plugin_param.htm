Задание параметров для плагина

 Top  Previous  Next

Закладка <Parameters> - предназначена  для задания параметров, передаваемых в плагин и получаемых из плагина (если необходимо).

 

Тип параметра может быть LONG - число или STR - строка.

 

Направление передачи параметров - IN - входящий параметр, OUT - исходящий параметр, INOUT - в обе стороны.

 

Параметр с номером 0 всегда передается в плагин (даже если нет других параметров) и может использоваться для получения возвращаемого значения (например, для кода ошибки).

 

После выполнения работы плагина возвращаемая информация может быть получена двумя способами (зависит от функций плагина):

1.         из стандартного источника <Record> - запись с неограниченным числом полей, в которой поля нумеруются по порядку (от 0), при этом поле с номером 0 - возвращаемое значение параметра 0 - всегда тип LONG, а все остальные поля могут быть типа LONG или STR. (например, плагин MSSQL.pvt возвращает данные в <Record>-записи после выполнения SQL-запроса SELECT) 

2.        через OUT или INOUT <Variable

 

 

·<Insert> - кнопка для добавления параметров (вставка после выбранного);
·<Delete> - кнопка для удаления выбранного параметра;
·<DeleteAll> - кнопка для удаления всех параметров;
·<№> - номер параметра;
·<Type> - тип параметра: STR - строка, LONG - число;
·<In/Out> - направление передачи параметра: IN - входящий параметр, OUT - исходящий параметр, INOUT - в обе стороны;
·<Source> - стандартный источник данных;

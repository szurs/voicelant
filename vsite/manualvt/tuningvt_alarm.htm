Tab <Alarm> - setting options for Alarms

·<Do alarm> - if checked, then upon the occurrence of significant events (such as accidents or mistakes) in a special directory (see below) put the files with the description of the event. Types of events:
1.Critical – critical events that affect the functioning of the system as a whole.
2.Not critical – events that affect the functioning of the system in the processing of certain functions or the reasons of such events are temporary.
3.Warning – warning of the need to produce action on the maintenance of the system.
4.Notice – Notification of system events or restore functionality.
5.Working – work events.
6.Impotant  - important events.

File Format event: LEVEL: type of event, HOST: name of the computer that sent the event, SOURCE: the source of the event (system modules), DATE: date of the event, TIME: time of the event, ALARM: description of the event.

·<Directory for alarm events> - Enter subdirectory for files of events (usually. \ ALARM). Button to the right - to select the subdirectory.

Logical container of service

 Top  Previous  Next

When creating new services is recommended to pack them in a logical container (service box).

 

Logical container of service - this is a sub-directory of file system, which contains all messages (voice and fax files), and auxiliary files needed at work and scenario for processing of this service.

 

Messages of service - files formats are WAV or TIF, ie sound or fax. They have digital names (for example, from 000001.wav to 999999.tif). The first message of service - the title, always has a name 0.WAV.

To set the directory container has a command -<Set - Work Directory>. To set the size of container  - <Set - Capacity of Box.>

 


Voicelant has special data types, which move the indicator (pointer) inside the container services:

BEGIN - move on to the title;

END - go to last message of service;

NEXT - go to next message;

BACK - to move to the previous message;

POINT - current message;

Handling these types of data produced by different commands, depending on the context, for example, the command with data type "BEGIN" plays the sound of the title 0.WAV, and the next command  with data type "NEXT" plays sound (or displays fax) from a file, the next in order for the title.

 

You also can play messages under their numbers in a service box.

 

Command writes the message in place, that is an indicator, the order of messages is not violated

 

Command  deletes the message, that is an indicator (the title will never be removed, but it can be rewritten).

 

Sound files in a container with non-numerical names may be used as an auxiliary in the formation of the response of the service subscriber.

 

Scenario files (.PAT) are used to specify the logic of this service.

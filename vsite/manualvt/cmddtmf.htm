Принять тональный набор (DTMF)

 Top  Previous  Next

Команда <DTMF> принимает из линии символы (цифры) тонального набора (DTMF - Dual Tone Multi Frequency: 1 2 3 4 5 6 7 8 9 0 # * A B C D) и сохраняет их в переменной DTMF.

 

 

·<Count Digits> - количество принимаемых символов (по умолчанию - 1, до 32);
·<Duration, sec> - таймер ожидания символов DTMF из линии (при принятии каждого символа таймер стартует снова, может = 0, по умолчанию = 8 сек);
·<Terminator> - задание завершающего символа последовательности (может не быть, по умолчанию - #). При принятии заданного символа-терминатора ввод завершается, т.е. можно вводить переменное число символов до терминатора. Символы терминаторов "#" и "*" имеют особый статус - если перед терминатором приняты какие-либо цифры DTMF, то эти терминаторы не сохраняются в переменной DTMF.

Обычно принятие тонального сигнала DTMF прерывает чтение () или запись (), символы сохраняются во внутреннем буфере программы и, далее выполнение сценария продолжается до команды <DTMF>, которая копирует данные из внутреннего буфера в переменную DTMF. Если таймер = 0, то выполение сценария продолжится сразу же. Далее можно проверить переменную DTMF и сравнить ее с помощью команды с разными значениями.

 

Условия выполнения - если приняты какие-либо символы DTMF - , если таймаут - .

 

Команда - <Detect DTMF> влияет на возможность принятия сигналов DTMF. Если установлено Detect DTMF=ALL (по умолчанию), то принимаются все символы DTMF, если  Detect DTMF="символу терминатора" (например, Detect DTMF="#"), то программа будет реагировать только на ввод терминатора.

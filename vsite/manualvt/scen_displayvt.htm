Проверка (прогон) сценария - DisplayVT

 Top  Previous  Next

Проверка сценария, созданного с помощью VisualVT, производится с помощью программы DisplayVT, которая может запускаться как из оболочки VisualVT, так и отдельно.

 

При проверке осуществляется прогон сценария, максимально приближенный к фактическому вызову при установленном соединении. Перед началом прогона подключаются все плагины, можно ввести дополнительные данные вызова. Речевые файлы воспроизводятся через звуковую карту, пользователь может ввести сигналы DTMF с помощью цифровой клавиатуры. Таким образом можно настраивать работу услуг без подключения их к работающей системе.

 

Если DisplayVT запускается как отдельная программа, то в верхней строке редактирования можно ввести сценарий или загрузить его из файла , но обычно запускается из приложения VisualVT - кнопка (Run).

 

· <DNIS> - ввести набираемый номер (заносится в переменную DNIS);
· <ANI> - ввести номер звонящего (заносится в переменную ANI);
· <REDIRNUM> - ввести номер переадресации (заносится в переменную REDIRNUM, только для ОКС7);
·<Play Messages> - если отмечен, то сообщения проигрываются через звуковую карту, если нет - только визуальный вывод;
·<NumPad> - клавиатура обычного телефонного аппарата, используется для ввода сигналов DTMF;
·<Start> - запуск прогона;
·<Stop> - остановка прогона;

 

Ограничения:

1.Не набирает номер и не устанавливает соединения.
2.Не понимает факсов.
3.Не воспроизводит сигналы DTMF
4. Не записывает сообщения.
5.Не выполняет команды ,

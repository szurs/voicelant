Tab <Log> - setting categories of events in the trace.

· <Log into File "application(t or f).log"> - if checked, the trace is written to the file applicationt.log or into the file applicationf.log, when the maximum size (~ 30 Mb) is reached. The information is then stored in them alternately;
· <View system messages > - output tracing of system events (start program, start service, etc.) ;
· <View calls messages>tracing output on the init and completion of incoming and outgoing calls;
·<View plugins messages> - Plugins events;
· <View requests messages> - tracing passing internal requests (for example, to plug-ins);
· <View incall messages> - tracing passing messages within call (for example, DTMF);
· <View FAX messages> - tracing facsimile functions;
· <View incall events> - tracing passing of internal events ;
· <View call's states> - tracing states of call;
· <View incall functions> - tracing calls internal functions (for example, "read file");
· <View pattern> - tracing commands of scenario;

When you create a script using VisualVT need to know some features of organization and rules:

 

1. Scenario - a program written in a script-like language, and presented visually in a tree in the program VisualVT. Executing the script is top down.
2.The elements of language - commands, that can take data from standard sources and after returning them to the variables and records. All commands are elementary in the sense that perform only the final action.
3.Standard sources of data this is sort of a finite set of types of "containers" containing the necessary information. For different sets of commands specific data sources may be different.
4. For presentation of data uses two types - text (STR) and numeric (LONG). The main type - text. Plugins need to specify a particular type and direction of data transmission.
5.Script commands can be grouped by limiting their braces ({ and }), see ). This final piece of the script can be written in a file with context menu (<Save to file + Include>) and further connected to the script as necessary by command . Nested parentheses is not limited.
6.Branching in the scenario are organized in terms of ( - Successfully , - Fails, always in pairs), which are checked after executing commands. If condition True - then all script commands inside the curly brackets are executed before . If condidion False - - then run all commands in curly brackets or one command after .
7.A command is intended for a choice from various variants of branching. It compares data of key (obtained from standard sources) with the variants of choices and executes the script corresponding coincidence.
8.Cycles in the scripts are organized within the braces, which comprises a group of commands. The number of repetitions of the cycle can be obtained from standard sources. Cycle with the number of repetitions equal to 1 - it's just braces. Exit from the cycle - to achieve the number of repetitions or by command
9.Files used to create scenarios, grouped by type and placed in the appropriate subdirectories:
· \Serv_Mes - working library of audio files of system;
· \DUTY - library files for reading numbers and currency values;
· \DATE - library files for reading dates;
· \TIME - library files for reading time;
· \CHAR - a library of sound files of characters;
· \PATTERN - files of scenarios;
· \PLUGINS - files installed plug-ins;
10.If you plan to provide many different types of services, it is recommended to pack them in logical containers.

               

 

Sample script that allows you to listen to 5 times the data on wages of employees by their codes, entered a touch-tone method. Extracted from the database on SQL - query. If a symbol # is entered, then the call is completed.

 

Consider the script line by line:

1.< Loop count = 5> (see ) - a cycle in the curly brackets, the number of repetitions of the cycle - 5;
2. < Read message from file "EnterCodeWorker.wav"> (see ) - reading file EnterCodeWorker.wav from subdirectory \Serv_Mes;
3. < Receive 2 DTMF Digits with Duration 8 seconds and Terminator = "#"> (see ) - receive 2 DTMF characters, expecting every 8 seconds to complete the input symbol #. You can enter before two symbols as code of employee but if you enter terminator # then it will be deleted from string of enter and cycle will be completed. Symbols are stored in the variable DTMF.
4. < Digits are received> - the condition is successful: received 2 digits DTMF, or the symbol "#".
5. Оpen Brace> - opening brace, limiting the group of commands that must be met if the condition was successfully , i.e. digits DTMF are received.
6. < Is variable "DTMF" equal text "%#%"> (see ) - A comparison of the variable DTMF (received characters) with a string containing a wildcard characters. As wildcard symbols you can use: "%" - to replace any number of characters, "_" - to replace one character (in the Windows "*" and "?", But "*" is on the telephone keypad). Here we check whether the entered symbol "#".
7. < Yes> - yes, the symbol "#" is present in variable DTMF.
8. <Оpen Brace> - opening brace for the commands that are executed, if was entered character "#".
9. < Break current loop> (see ) - exit from current cycle and move to string № 32.
10. < Close brace> - closing bracket for line number 8.
11. < - No> - condition is false, i.e symbol "#" wasn't received and characters (two digits) of employee's code were received.
12. <Оpen Brace> - opening brace for the commands that are executed, if condition is false , i.e. process data in a variable DTMF
13.< Run query from text "select * from workers where number = ?" in database "MSSQL"> (see )  - Query the database, connected via the plug-in "MSSQL", where instead of ? substitutes the value of DTMF. Request written on SQL language.
14.< Success> - database query is executed successfully and in record <Record> are data of Recordset after completed SQL - query "select...";
15.<Оpen Brace> - opening brace for the commands that must be run if SQL - query is completed well.
16.< Read message from file "CodeWorker.wav"> (see ) - reading file CodeWorker.wav from subdirectory \Serv_Mes ("Your code employee..."). Reading can be interrupted by entering any symbol DTMF, after that run of scenario is continued and will go to string 3 by cycle (or call terminated);

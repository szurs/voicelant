Типы источников данных

 Top  Previous  Next

Для получения информации командами задано конечное число типов стандартных источников данных.

Для разных команд набор источников данных может меняться.

 

·<File> - информация содержится в файле, котрый необходимо указатьв строке ввода или выбрать кнопкой. Для разных команд типы файлов могут быть разными, например, для команды <Play file> - это файлы с расширениями WAV или TIF, для команды - текстовые файлы TXT, а для команды - файлы сценариев PAT.

 

·<Text> - информация представлена непосредственно в виде текста. Если размер текста небольшой, например, номер телефона, то строку можно ввести прямо в поле ввода, если размер текста значительный и в нем присутствуют символы перевода строки, то можно ввести нужный текст в простом редакторе, который вызывается кнопкой, в нем же можно загрузить текст из файла. Очень большие тексты желательно хранить в файлах.

· <Record> - информация находится в записи, которая может содержать неограниченной число полей. Запись обычно используется для получения информации из программного модуля - плагина, когда неудобно или невозможно передавать данные через переменные, передаваемые в плагин. Например, при выполнении SQL - запроса "select * from table" к плагину, поддерживающему соединения с базой данных, результаты возвращаются в виде , в котором число полей может быть большим или неизвестным. Доступ к полям записи можно получить по порядковому номеру поля набором нужного числа: , или получить всю запись целиком: для сохранения в переменной. Поле с номером 0 используется для получения возвращаемого значения из плагина, например, результат выполнения или код ошибки.

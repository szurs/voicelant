Задать переменную (Variable)

 Top  Previous  Next

Команда <Variable> создает новую переменную либо перезаписывает старую значением, полученным из стандартных источников:

 

 

·Enter or select name of variable - ввести новое имя переменной или выбрать старое;

 

Задать значение переменной можно из стандартных источников:

 

 

·<File> - ввести или выбрать кнопкой имя файла, в котором содержится новое значение переменной;
·<Text> - ввести новое значение переменной;
·<Variable> - ввести или выбрать имя переменной из которой берется новое значение (копирование переменных);
·<Record> - набрать номер поля записи, содержащего задаваемое значение переменной (или всего массива (array),т.к. переменной для хранения можно присвоить массив (см. команду ));
·<Counter> - текущий счетчик цикла;
·<Info BOX> - выбор информации о текущем контейнере услуги (если ранее был задан):
1.Directory - строка пути до подкаталога услуги;
2.Volume (sec) - текущая наполненность ящика услуги в секундах;
3.Volume (min) - текущая наполненность ящика услуги в минутах;
4.Balanse (sec) - остаток емкости ящика услуги в секундах;
5.Balanse (min) - остаток емкости ящика услуги в минутах;
6.Number vox - количество звуковых сообщений в ящике;
7.Number fax - количество факсимильных сообщений в ящике;
·<Date> - текущая дата (в формате "DD-MM-YYYY");
·<Time> - текущее время (в формате "HH:MM:SS");

 

Список заранее определенных переменных, которые изменяют свое значение в зависимости от состояния вызова:

·ANI (CALLING_NUMBER)- хранит номер звонящего (вызывающего) абонента
·DNIS (CALLED_NUBER) - телефонный номер пункта назначения (вызываемый номер).
·DTMF - цифры, набранные абонентом тональным способом.
·REDIRNUM (REDIRECTING_NUMBER) - номер, с которого был переадресован вызов (зависит от типа сигнализации).
·CALL_ANALYSIS (DISC_CAUSE) - текстовые значения состояния вызова после набора номера. Анализируются сигналы в линии. Возможные значения:
1.BUSY - занято.
2.NOANSWER - нет ответа.
3.NORINGBACK - нет сигнала КПВ.
4.CONNECTED - соедиенние установлено.
5.INTERCEPT - прерывание оператором.
6.STOPPED - остановлено
7.NODIALTONE - нет ответа станции.
8.FAXTONE - ответил факс или модем.
9.ERROR - ошибка
·TYPE_CONNECT - тип установления соединения. Возможные значения:
1.CADENCE - по заданному шаблону (например, по пропаданию сигнала КПВ)
2.LOOPCURRENTDROP - по переполюсовке в линии.
3.VOICE - по голосовому ответу.
4.ANSWERINGMACHINE - по ответу автоответчика.
6.PBX - учрежденческая АТС.
·TERM_COND - условия завершения голосовых команд
·NAME_LASTFILE - имя последнего записанного или прочитанного голосового файла
·OFFSET_LASTFILE - смещение в байтах от начала файла на котором прервалось чтение или запись последнего файла.
·CALLING_CATEGORY - категория абонента (зависит от типа сигнализации).
·OUTCALL_RESULT - результат исходящего вызова.
·REMOTE_CALLING_NUMBER
·REMOTE_CALLED_NUMBER
·REMOTE_REDIRECTING_NUMBER

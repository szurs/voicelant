Setting Plugins

Tab <Plugins> - Configure installed plugins.

To configure the selected plugin double click on it with the mouse.

Plug-ins are software modules that empower Voicelant. They are used for transmitting, receiving, processing and data conversion. Set of plugins for the system depends on the specific tasks. Additional plugins can be connected when you need. Basic plug-ins included in the delivery Voicelant, others are supplied on order of the customer.

Plugin - this is a file with the extension .pvt, is developed on the specification of interface plug-ins.

·Installing the plugin  - this is placing the file of plugin in subdirectory .\PLUGINS system Voicelant.
· Connecting the plugin is carried out when the program starts.
· Calling the plug-in occurs from the scenario by name of service in an operating time.

 

Plug-ins are standard and nonstandard. Standard plug-ins can have a file <name_plugin>.ini which contains the basic settings the plug-in:

· <Path Plugin> - path to the file plug-in.
· <Service name> - name of the service to call the plugin from the scenario while working. Some plugins (for example, MSSQL.pvt) may have several services with different names (MSSQL and MS-SQL) and the parameters, for example, to split queries to different data sources with different response time.
· <Connection string> - additional line parameters used when connecting plugin (can be empty);
· <Number of threads> - the number of software threads that can simultaneously handle requests to this service. For example, for the plugin MSSQL.pvt and service name MSSQL we will have two threads for connection with MS-SQL Server to reduce response time.

If the standard plug-in does not have file <name_plugin>.ini, it is understood that this plugin uses only the internal settings and gets only one thread for executing.

 

Non-standart plug-ins within itself contain a window (dialog) to set the necessary specific parameters, which is called as well as for the standard plug-ins.

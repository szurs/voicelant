Особенностью Voicelant является возможность подключения дополнительной функциональности через использование программных модулей, написанных сторонними поставщиками - плагинов. Они используются для передачи, получения, обработки и преобразования данных. Набор плагинов для системы зависит от конкретных задач. Дополнительные плагины можно подключать по мере небходимости. Основные плагины входят в поставку Voicelant, другие поставляются по желанию заказчика. Такие команды Voicelant, как , , реализованы с помощью плагинов, входящих в комплект поставки. 

Примеры плагинов для расширения - различные способы доступа к базам данных (OLE DB, ODBC, фирменные интерфейсы), HTTP, FTP, Telnet доступ к данным, отправка и получение e-mail,, проприетарные интерфейсы, нестандартные источники данных (редкие БД, системы промышленной автоматизации - датчики). При развитой системе плагинов Voicelant может даже не использовать телефонную часть в своей работе, а заниматься только обработкой и диспетчеризацией данных.

Плагин - это файл с расширением .pvt, разработанный с учетом спецификации интерфеса плагинов,

·Установка плагина  - это помещение файла плагина в подкаталог .\PLUGINS системы Voicelant.
·Подключение плагина осуществляется  при запуске программы Voicelant.
·Вызов плагина происходит из сценария по имени сервиса во время работы Voicelant с помощью команды .

 

Плагины бывают стандартные и нестандартные. Стандартные плагины могут иметь файл <имя_плагина.ini в котором содержатся основные параметры настройки плагина (см. Настройка параметров плагинов:

·<Path Plugin> - путь до файла плагина.
·<Service name>вызова плагинаVoicelant. Некоторые плагины (например, MSSQL.pvt) могут иметь несколько сервисов с разными именами (MSSQL и MS-SQL) и параметрами, например, для того, чтобы разделить запросы к различным источникам данных, имеющим разное время отклика;
· - дополнительная строка параметров, используемая при подключении плагина (может быть пустой);
·<Count threads> - количество программных потоков, которые одновременно могут обрабатывать запросык данному  сервису. Например, для плагина MSSQL.pvt - это количество соединений с базой данных MS-SQL Server одновременно поддерживаемых при работе Voicelant для сокращения времени отклика.

Если стандартный плагин не имеет файла <имя_плагина.ini, то подразумевается, что данный плагин использует только внутренние настройки и получает при подключении только один поток для выполнения.

 

Нестандартные плагины внутри себя содержат окно (диалог) для настройки необходимых специфических параметров, которое вызывается так же как и для стандартных плагинов.

 Voicelant-AISST: automatic outbound notification and inbound information system

Voicelant-AISST - a simple telephone system, which combines the functions of automatic outbound notification and automatic inbound information system. Connected to the analog telephone lines.

As outbound notification system, AISST help solve problems associated with the need to inform any number of subscribers of the telephone network to transfer them voice messages in the speech form.

Algorithm of notification sysnem is defined by a script written in simple script-like language which allows to build any sequence in advance dictate phrases, numerical data, the amount of debt, phone numbers, date and time.

By changing the scenario, you can create other algorithms warning, which would be best to meet your needs.

The source of data for AISST may be a text file or a database, access to which is done via ODBC.

Calls made for a certain period of time as scheduled.

At the same time can be made a few outgoing calls by number of channels. Each call is serviced by own scenario and works under own schedule.

Numbers notified by the computer are marked.
If the subscriber is busy or not responding, the phone number is not marked as notified, the computer dials the next number in the order in a loop until all the subscribers will not be notified.

The protocol of the notified subscribers is kept.

In the supply AISST included examples:

  • Notification of Debt
  • Notification of the temporary switch off the phone
  • Alert arbitrary voice
  • Delivery of messages to confirm reception
  • Delivery of messages with the record of the call has been connected
  • Reminder to pay loan
  • Alert Emergency Services

As a inbound information system, AISST would allow an automated help desk, which responds to calls of subscribers and give them voice messages in the form of background information.

Different phone numbers you can assign different services, each of which operates on a script written in a simple script-like language.
You can expand the number of services, if consider the caller identification by CallerID, or by DTMF dialing.

The data for the information can be obtained from a text file or from any database via ODBC.

A protocol of incoming calls, which is recorded all the information about the calls.

In the supply AISST included examples of services:

  • Information about debt by CallerID
  • Information about debt by DTMF
  • Information on Tariffs
  • Help on the balance of funds in the account
  • Weather
  • Local Time

AISST based on multi-channels (2, 4 or 12) voice cards, installed in a free slot of a PC with OS Windows 2000/2003/XP and connects to a corresponding number of subscriber telephone lines.

The operator can adjust each channel separately, which allows flexibility to combine different services. For example, if you purchased 4 - channel board, the operator may include one phone line to "telephone calls about the debt", second line used to provide customers information on the time and the remaining two lines - to provide information about the weather.

If you wish to have a system with far greater capabilities and scalability, see Voicelant-IVR.

The number of analog telephone lines, which connects Voicelant-AISST is determined by the customer and may be 4, 8 or 12 channels (or more, but then better to use a digital version of Voicelant-IVR).

Voicelant-AISST can be supplied in various ways:

  • "box" version: Dialogic board with the necessary number of channels and Voicelant-AISST;
  • only software Voicelant-AISST;

The cost of the system is determined by the number of channels.
